## Plugin Philosophy

Plugins in Librarian follow these core principles:

1. **Single Responsibility**: Each plugin handles one type of content analysis
2. **Stateless Processing**: Plugins process content and generate metadata without maintaining state between runs
3. **Deterministic Output**: Same input always produces same output (for metadata extraction)
4. **Explicit Configuration**: Plugins are enabled/disabled via configuration, not hardcoded

## Plugin Lifecycle

```
File
  ↓
Plugin
  ↓
Metadata
  ↓
Database
  ↓
Search Index
  ↓
User Query
```

### Lifecycle Stages

| Stage | Description |
|-------|-------------|
| **File** | Document enters the system via discovery |
| **Plugin** | Plugin processes the file based on its artifact type |
| **Metadata** | Plugin generates structured metadata |
| **Database** | Metadata is persisted to the appropriate table |
| **Search Index** | Metadata is indexed for retrieval |
| **User Query** | User searches and retrieves results |

---

## Plugin Responsibilities

Each plugin is responsible for:

1. **Input Validation**: Verify the file type and structure are supported
2. **Content Processing**: Extract or analyze the file content
3. **Metadata Generation**: Create structured, searchable metadata
4. **Error Handling**: Gracefully handle corrupt or unsupported files

| Plugin | Artifacts |
|--------|-----------|

| Plugin | Searchable Metadata |

## Plugin Configuration

Plugins are configured in `config/plugins.yaml`:

```yaml
plugins:
  photo_metadata:
    enabled: true
    job_type: extract_photo_metadata
  thumbnail:
    enabled: false
    job_type: generate_thumbnail
  ocr:
    enabled: false
    job_type: run_ocr
  object_detection:
    enabled: false
    job_type: object_detection
```

## Plugin Registry

The `registry/plugin_registry.py` module provides plugin management:

- Enable/disable plugins at runtime via environment variables
- Validate plugin/handler alignment at startup
- Prevent scheduling of jobs without registered handlers

See [P13-P15 Plugin Architecture](../refactor/P13-P15-plugin-architecture.md) for implementation details.

---

## Creating a New Plugin

To add a new plugin:

1. Create `docs/plugins/<plugin-name>/` directory structure
2. Add `README.md`, `capabilities.md`, `metadata-schema.md`, `architecture.md`, `roadmap.md`
3. Register the plugin in `config/plugins.yaml`
4. Implement the worker handler in `workers/`
5. Update `registry/plugin_registry.py` if needed
